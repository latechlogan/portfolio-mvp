Interactive Hero Slider

Replacing a performance-heavy video background with responsive, multi-CTA storytelling

Role Design & Development
Tools HTML, CSS, JavaScript
Context Healthcare Website Redesign

The Problem

Our homepage hero section had issues:

  • Video background caused performance problems on slower connections
  • Had to disable video entirely on mobile, replacing it with a static image
  • Could only accommodate one call-to-action
  • Stakeholders needed to promote multiple campaigns in that prime real estate

Before: Single CTA with video background (disabled on mobile)

What I Built

An auto-advancing slider with story-style navigation that:

  • Works smoothly across all devices without performance issues
  • Lets stakeholders showcase multiple CTAs in a clean interface
  • Gives users control with clickable progress bars and navigation arrows
  • Scales text independently from graphics for readability at any screen size
  • Provides visual feedback so transitions feel intentional, not jarring

After: Multi-CTA slider with story-style navigation

Technical Challenges

  • Responsive text: Separated text from graphics so it could scale properly at any screen size—embedding text in images would make it illegible on small screens or oversized on large ones
  • Image scaling: Used CSS aspect-ratio to let graphics resize fluidly without needing multiple image sizes or restrictive safe zones
  • Accessible navigation: Included arrow buttons alongside the progress bar interface for users who might not be familiar with story-style navigation (our audience includes seniors)

How This Came Together

  • Worked as solo dev on a small marketing team managing three healthcare sites
  • Stakeholders wanted multiple CTAs, I knew the video needed to go—goals aligned perfectly
  • Moved straight from requirements to code (our team prioritized shipping over mockups)
  • Secured buy-in from marketing director and business owners before launch

Results

  • Eliminated performance issues across all devices
  • Delivered the multi-CTA functionality stakeholders needed
  • Created a seamless responsive experience from mobile to desktop
  • Received zero negative user feedback (notable given our diverse age demographic)
  • Got enthusiastic stakeholder approval

Note: I actively advocated for analytics tracking to validate UX decisions with data, but we didn't have that infrastructure in place at the time.
